▸ run(rules)

Runs this filter with the specified set of rules. Rules are an array of arrays containing objects describing the rules for this filter. Each object has the following structure: { name: string - The name of the column this rule applies to value: * - The value to compare the column's values with operation: string - Which operation to perform, must be one of the following: "equal", "notEqual", "lessThan", "moreThan" or "contains" }

Rule objects within the same array are treated as ANDed and separate rule arrays are ORed. Here's an example of a valid rules array:

[
[
{
name: 'Origin_airport',
value: 'SEA',
operation: 'equal',
},
{
name: 'Destination_airport',
value: 'LAX',
operation: 'notEqual',
},
],
[
{
name: 'Origin_airport',
value: 'MCO',
operation: 'equal',
},
{
name: 'Passengers',
value: 180,
operation: 'moreThan',
},
],
]

In the example above, the filter requires that in the results: { The value of column Origin_airport is equal to SEA AND The value of column Destination_airport is not equal to LAX } OR { The value of column Origin_airport is equal to MCO AND The value of column Passengers is more than 180 }

run(rules: Array<Array<Object>>): Promise<{memory: MemoryBlock, count: number}>
Parameters
rules (Array<Array<Object>>) The rules to run this filter with.
Returns
Promise<{memory: MemoryBlock, count: number}>:
